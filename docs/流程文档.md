```

    │

    │   └─ 飞书文档 URL
    │
    ├─ 提供代码（可选）
    │   ├─ 上传 ZIP 压缩包
    │   └─ 填写本地目录路径
    │
    ▼
点击「生成测试用例」
    │
    ├─ 解���文档 → 提取需求文本
    ├─ 分析代码 → 生成代码结构摘要（可选）
    ├─ 调用大模型 → 生成测试用例 JSON
    ├─ 导出 Excel 和 Markdown 文件

```

## 二、详细步骤说明

### 步骤 1：配置大模型

> 首次使用前必须完成此步骤。

1. 点击导航栏「配置」
2. 选择提供商（OpenAI / Anthropic / 自定义）
3. 填写 Base URL、API Key、模型名称
4. 如需飞书集成，填写 App ID、App Secret
5. 点击「保存配置」

**常用模型参考：**

| 提供商 | 推荐模型 |
|--------|---------|
| OpenAI | `gpt-4o`、`gpt-4-turbo` |
| Anthropic | `claude-sonnet-4-20250514` |
| 自定义 | 按实际填写 |

---

### 步骤 2：上传需求文档

支持以下方式之一：

**方式 A：上传本地文件**
- 支持格式：`.docx`、`.pdf`、`.md`、`.txt`
- 文件大小限制：50 MB
- 建议使用结构清晰的需求文档（含功能描述、业务规则等）

**方式 B：飞书文档 URL**
- 格式示例：`https://xxx.feishu.cn/docx/AbCdEfG`
- 需在配置页填写飞书 App ID 和 App Secret
- 支持飞书文档（docx）和知识库（wiki）

---

### 步骤 3：提供代码（可选）

提供代码可显著提升测试用例的针对性和覆盖率。

**方式 A：上传 ZIP 压缩包**
- 将项目源码打包为 `.zip` 上传

**方式 B：填写本地目录路径**
- 在服务器可访问的前提下，填写绝对路径，如 `/Users/xxx/myproject`

---

### 步骤 4：生成测试用例

点击「生成测试用例」按钮后：

1. 系统解析文档，提取需求文本
2. 如有代码，扫描目录结构并生成摘要
3. 将需求 + 代码摘要组装为 Prompt，调用大模型
4. 解析大模型返回的 JSON，结构化为测试用例列表
5. 自动生成 Excel 和 Markdown 导出文件
6. 如配置飞书，自动创建飞书文档
7. 跳转至结果页

> 生成时间根据文档大小和模型速度，通常为 10~60 秒。

---

### 步骤 5：查看与导出结果

结果页功能：

| 功能 | 说明 |
|------|------|
| 按模块筛选 | 下拉选择功能模块 |
| 按优先级筛选 | P0~P3 过滤 |
| 按类型筛选 | 功能/UI/边界/性能/安全/集成 |
| 关键词搜索 | 实时搜索用例名称和步骤 |
| 下载 Excel | `.xlsx` 格式，含格式化表头 |
| 下载 Markdown | `.md` 格式，适合粘贴到文档 |
| 飞书文档 | 点击链接直接跳转飞书 |

---

### 步骤 6：优化与重新生成（可选）

如对生成结果不满意，可在结果页输入反馈：

```
示例：
- 请补充登录失败的边界用例
- 支付模块的安全测试用例太少
- TC-005 的步骤描述不够详细
```

系统将在原有用例基础上，根据反馈修改和补充，重新生成完整列表。

---

### 步骤 7：生成自动化测试脚本（可选）

1. 在结果页点击「生成测试脚本」
2. 输入项目代码路径（支持自动识别前端/后端）
3. 系统分析代码结构，生成对应的自动化测试框架代码
4. 下载生成的测试项目

---

## 三、历史记录

- 每次生成结果自动保存到本地（保留1小时）
- 点击导航栏「历史记录」可查看所有历史生成记录
- 点击具体记录可重新查看和下载

## 四、常见问题

### Q：生成失败，提示 API 错误
- 检查配置页的 API Key 是否正确
- 确认 Base URL 格式正确（OpenAI 需以 `/v1` 结尾）
- 确认模型名称拼写正确

### Q：飞书文档 URL 解析失败
- 确认 URL 格式为 `https://xxx.feishu.cn/docx/TOKEN`
- 确认飞书 App 已被授权访问该文档
- 检查 App ID 和 App Secret 是否填写正确

### Q：代码分析结果不理想
- 建议只上传与需求相关的核心模块代码
- 避免上传含大量第三方依赖的目录
- 可先在本地删除 `node_modules`、`.venv` 等目录再打包

### Q：生成的测试用例数量太少/太多
- 在反馈框中明确说明期望数量或覆盖范围
- 例如：「请至少生成 50 条用例，重点覆盖异常场景」

### Q：测试用例导出文件在哪里
- 页面直接点击下载按钮即可获取
