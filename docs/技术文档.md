# TestCraft 技术文档

## 一、技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Python / Flask |
| AI 集成 | OpenAI / Anthropic / 自定义兼容 API |
| 文档解析 | python-docx、PyPDF2、Markdown |
| 代码分析 | 自研 code_analyzer（支持 ZIP 压缩包和本地目录） |
| 导出 | openpyxl（Excel）、Markdown |
| 飞书集成 | 飞书开放平台 API |
| 前端 | Jinja2 模板 + 原生 JS + CSS |

## 三、核心模块说明

### 3.1 doc_parser.py — 文档解析

支持以下输入格式：

| 格式 | 处理方式 |
|------|---------|
| `.docx` | python-docx 提取段落和表格 |
| `.pdf` | PyPDF2 提取文本 |
| `.md` / `.txt` | 直接读取 |
| 飞书文档 URL | 调用飞书 API 获取文档内容 |

飞书 URL 格式支持：
- `https://xxx.feishu.cn/docx/TOKEN`
- `https://xxx.feishu.cn/wiki/TOKEN`

### 3.2 code_analyzer.py — 代码分析

- 接收 ZIP 压缩包路径或本地目录路径
- 扫描目录结构，识别主要文件和模块
- 输出 `CodeStructure` 对象，调用 `.to_text()` 转换为文本供 LLM 使用
- 忽略 `node_modules`、`.git`、`__pycache__` 等无关目录

### 3.3 ai_generator.py — AI 生成

- 将需求文本 + 代码结构拼装为 Prompt
- 调用配置的大模型 API（`call_llm()`）
- 解析返回的 JSON 数组（`_parse_response()`）
- 返回标准化测试用例列表

**测试用例字段：**

| 字段 | 说明 |
|------|------|
| id | 唯一标识（TC-001...） |
| module | 功能模块 |
| name | 测试用例名称 |
| priority | P0（关键）~ P3（低） |
| preconditions | 前置条件 |
| steps | 测试步骤 |
| expected_result | 预期结果 |
| type | 类型（功能/UI/边界/性能/安全/集成） |

### 3.4 export.py — 导出

- **Excel**：使用 openpyxl，含表头格式化，输出 `.xlsx`
- **Markdown**：输出标准 Markdown 表格格式

### 3.5 feishu_writer.py — 飞书集成

- 使用飞书开放平台 API 创建文档
- 需配置 `feishu_app_id`、`feishu_app_secret`��`feishu_domain`
- 返回创建的文档 URL

### 3.6 test_script_gen.py — 测试脚本生成

- 根据测试用例 + 代码路径，调用 LLM 生成可执行测试脚本
- 支持前端（React/Vue 等）和后端（Python/Go 等）代码路径
- 输出测试架构说明和脚本文件

## 四、API 接口列表

### 页面路由

| 路由 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 首页（上传页） |
| `/configure` | GET/POST | 大模型配置 |
| `/generate` | POST | 生成测试用例 |
| `/results` | GET | 测试用例结果展示 |
| `/download/<fmt>` | GET | 下载文件（excel/markdown） |
| `/history` | GET | 历史记录列表 |
| `/history/<run_id>` | GET | 历史记录详情 |
| `/run-test` | GET | 测试运行页 |
| `/autotest` | GET | 自动化测试页 |

### REST API

| 接口 | 方法 | 说明 | 请求体 |
|------|------|------|-------|
| `/api/create-feishu-doc` | POST | 创建飞书文档 | 无（使用 session 数据） |
| `/api/generate-scripts` | POST | 生成测试脚本 | `{"code_paths": [...]}` |
| `/api/regenerate` | POST | 根据反馈重新生成 | `{"feedback": "..."}` |
| `/api/provider-defaults/<provider>` | GET | 获取提供商默认配置 | 无 |
| `/api/detect-project` | POST | 自动识别项目代码路径 | `{"root": "/path/to/project"}` |

## 五、配置说明

### 大模型提供商配置（config.json）

```json
{
  "provider": "openai",
  "base_url": "https://api.openai.com/v1",
  "api_key": "sk-xxx",
  "model": "gpt-4o",
  "feishu_app_id": "",
  "feishu_app_secret": "",
  "feishu_domain": "https://open.feishu.cn"
}
```

### 支持的提供商

| 提供商 | Base URL |
|--------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com` |
| 自定义 | 任意 OpenAI 兼容 API |

### 环境变量

| 变量 | 说明 | 默认值 |
|------|------|-------|
| `FLASK_SECRET_KEY` | Session 加密密钥 | `testcraft-dev-secret-key` |

## 七、部署

### 本地开发

```bash
pip install -r requirements.txt
python app.py
# 访问 http://localhost:8899
```

### 生产环境建议

```bash
export FLASK_SECRET_KEY="your-secure-secret-key"
gunicorn -w 4 -b 0.0.0.0:8899 app:app
```

- 文件上传限制：**50 MB**
- 建议使用 Nginx 反向代理
- 建议配置定时任务清理 `uploads/` 和输出目录
